이슈 보안 계획에서 이슈 보안 등급을 생성하여 누가 이슈를 볼 수 있고 볼 수 없는지를 제어할 수 있습니다. 사용자나 그룹을 설정할 수 있는 보안 등급의 집합으로 이루어져 있습니다.

이슈를 생성하거나 수정할 때 해당 이슈의 보안 등급을 지정할 수 있습니다. 이렇게 하면 해당 이슈를 볼 권한이 있는 사람만이 볼 수 있게 할 수 있습니다.

이슈 보안 계획이 프로젝트와 연결 되며 해당 보안 등급은 해당 프로젝트의 이슈에 적용될 수 있습니다. 하위 작업은 부모 이슈의 보안 등급도 상속받습니다.

참고로, 연계된 프로젝트가 있다면 이슈 보안 계획을 삭제할 수 없습니다.

시작하기 전

  • 이슈 등급 보안을 설정하기 위한 JIRA 관리자 전역 권한으로 로그인합니다.
  • 이슈 등급 보안을 사용하려는 모든 사용자에게 프로젝트 별 '이슈 보안 설정' 권한이 있는지 확인합니다.

이슈 보안 계획 추가

  1. 선택 > 이슈 .
  2. 이슈 보안 계획 을 선택하여 이슈 보안 계획 페이지를 엽니다.
  3. 이슈 보안 계획 추가 를 클릭합니다.
  4. 요청 된 세부 사항을 기입하여 추가 를 클릭합니다.

보안 등급을 이슈 보안 계획에 추가

  1. 선택 > 이슈 .
  2. 이슈 보안 계획 을 선택하여 이슈 보안 계획 페이지를 엽니다.
  3. 이슈 보안 계획 이름이나 보안 등급 링크를 클릭하여 이슈 보안 등급 편집 페이지를 엽니다.
  4. 요청 된 세부 사항을 기입하여 보안 등급 추가 를 클릭합니다.

이슈 보안 계획의 기본 보안 등급 설정

이제 각 보안 계획에 할당 된 이슈에 적용될 기본 보안 등급을 선택할 수 있습니다.

기본 보안 등급을 설정할 때 유의해야 할 사항은 다음과 같습니다.

  • 이슈 보고자가 '이슈 보안 설정' 권한이 없는 경우 이슈는 기본 보안 등급으로 설정됩니다.
  • 이슈 보안 계획에 기본 보안 등급이 없는 경우 이슈 보안 등급이 '없음'으로 설정됩니다 (모든 사용자가 이슈를 볼 수 있음).
  1. 선택 > 이슈 .
  2. 이슈 보안 계획 을 선택하여 이슈 보안 계획 페이지를 엽니다.
  3. 이슈 보안 계획 이름이나 보안 등급 링크를 클릭하여 이슈 보안 등급 편집 페이지를 엽니다.
    1. 기본 보안 등급을 설정하려면 적절한 보안 등급 을 찾은 다음 작업 열에서 기본값 을 클릭합니다.
    2. 기본 보안 등급을 제거하려면 기본 보안 등급을 "없음"으로 변경 을 클릭합니다 (페이지 상단 부근) .

보안 등급에 사용자를 추가

  1. 선택 > 이슈 .
  2. 이슈 보안 계획 을 선택하여 이슈 보안 계획 페이지를 엽니다.
  3. 이슈 보안 계획 이름이나 보안 등급 링크를 클릭하여 이슈 보안 등급 편집 페이지를 엽니다.
  4. 보안 등급을 선택한 후 작업 열에서 추가 링크를 클릭하여 사용자/그룹/프로젝트 역할을 보안 등급에 추가 페이지가 열립니다.
  5. 적절한 사용자, 그룹 또는 프로젝트 역할을 선택한 다음 추가 버튼을 클릭합니다.

보안 등급의 구성원은 다음으로 구성 될 수 있습니다.

    • 개인 사용자
    • 그룹
    • 프로젝트 역할
    • '보고자', '프로젝트 리드' 및 '현재 담당자'와 같은 이슈 역할
    • '아무나' (예 : 익명 액세스 허용)
    • (멀티) 사용자 또는 (멀티) 그룹 선택기 사용자 정의 필드.

6. 모든 적절한 사용자, 그룹 또는 프로젝트 역할이 보안 등급에 추가 될 때까지 4 단계와 5 단계를 반복합니다.

프로젝트에 이슈 보안 계획을 할당

  1. 선택 > 프로젝트 클릭하여 관련 프로젝트를 선택합니다.
  2. 관심있는 프로젝트의 이름을 선택하여 프로젝트 요약 페이지가 표시됩니다.
  3. 프로젝트 요약 페이지의 사용 권한 섹션에서 이슈 레이블에 해당하는 링크를 클릭하여 프로젝트에 이슈 보안 계획을 연결 페이지를 엽니다. 이 이름은 프로젝트의 현재 이슈 보안 계획의 이름이거나 None 입니다.
  4. 이 프로젝트와 연결할 이슈 보안 계획을 선택합니다.
  5. 이전에 보안이 설정된 이슈가 없는 경우 (또는 프로젝트에 이전에 이슈 보안 계획이 없는 경우) 다음 단계를 건너 뜁니다.
  6. 이전에 보안이 설정된 이슈가 있는 경우 새 보안 등급을 선택하여 각 이전 등급을 대체합니다. 이전 계획의 보안 등급과 관련된 모든 이슈는 이제 새로운 계획의 보안 등급을 갖게됩니다. 이전에 보안이 설정된 모든 이슈에서 보안을 제거하려면 '없음'을 선택할 수 있습니다.
  7. 프로젝트를 이슈 보안 계획과 연결하려면 ' 연계 ' 버튼을 클릭합니다.

i. 이슈 등급 보안을 구성한 후 보안 등급 필드가 이슈의 화면에 표시되지 않으면 내 필드가 어디있지? 도구를 사용하여 표시되지 않는 이유를 확인합니다.

이슈 보안 계획 삭제

프로젝트와 연결된 이슈 보안 계획을 삭제할 수 없다는 것을 이해하는 것이 중요합니다. 먼저 JIRA 설치에서 이슈 보안 계획과 프로젝트 간의 연결을 제거해야 합니다.

  1. 선택 > 이슈 .
  2. 이슈 보안 계획 을 선택하여 이슈 보안 계획 페이지를 엽니다. 현재 JIRA 설치에서 사용 가능한 모든 이슈 보안 계획을 나열합니다.
  3. 삭제할 이슈 보안 계획의 작업 열에서 삭제 링크를 클릭합니다.
  4. 확인 페이지에서 삭제 를 클릭하여 삭제를 확인합니다. 그렇지 않으면 취소 를 클릭합니다.

이슈 보안 계획 편집

이슈 보안 계획의 이름과 설명을 편집할 수 있습니다. 또한 이슈를 편집 할 때 기본 보안 등급을 편집할 수도 있습니다.

  1. 선택 > 이슈 .
  2. 이슈 보안 계획 을 선택하여 이슈 보안 계획 페이지를 엽니다. 현재 JIRA 설치에서 사용 가능한 모든 이슈 보안 계획을 나열합니다.
  3. 편집할 이슈 보안 계획의 작업 열에서 편집 링크를 클릭합니다.
  4. 수정 한 다음 편집 을 클릭하여 편집을 확인합니다. 그렇지 않으면 취소 를 클릭합니다.

이슈 보안 계획 복사

  1. 선택 > 이슈 .
  2. 이슈 보안 계획 을 선택하여 이슈 보안 계획 페이지를 엽니다. 현재 JIRA 설치에서 사용 가능한 모든 이슈 보안 계획을 나열합니다.
  3. 복사할 이슈 보안 계획의 작업 열에서 복사 링크를 클릭합니다. 신규 이슈 보안 계획은 동일한 보안 등급과 동일한 사용자/그룹/프로젝트 역할로 할당됩니다. 신규 계획은 ' ... 의 사본 ' 이라고 할 것입니다. 원하는 경우 신규 계획을 편집하여 다른 이름을 지정할 수 있습니다.
