Oracle Java BCL 라이선스가 2018년 7월 부로 `Java SE subscription` 년단위 구독모델로 개편됐습니다.

Oracle 유료화 모델이 변경 되면서 Atlassian Jira, Software는 어떻게 대응하는지 알아봤습니다.

Oracle JDK는 유료인가?

2019년 1월 이후에 릴리즈되는 Oracle Java SE8은 유료 라이선스 없이 업무용, 상업용으로 사용할 수 없습니다.

Atlassian Jira, Confluence 설치할 때 JRE 1.8.181가 Bundle로 제공됩니다.

Java 1.8.181은 BCL 계약의 동의하에 무료로 사용할 수 있습니다.

BCL 계약에는 일반적인 컴퓨팅에서 사용할 수 있으며 Java를 수정하지 않는 범위에서 무료로 사용할 수 있습니다.

JDK(Java Development Kit)를 수정해서 사용하시면 구매를 하셔야합니다.

Java 자체는 오픈소스 이지만 Oracle Java는 무료 또는 유료라는 점!

Oracle은 OpenJDK를 OracleJDK와 동일하게 릴리즈합니다. Oracle의 기술지원 없이 OpenJDK로 라이선스 걱정없이 사용할 수 있습니다.

Atlassian Oracle Java 지원

Jira, Confluence는 Oracle Java가 필요합니다. 정확히는 JRE: Java Runtime Environment 가 필요합니다.

기존 Jira, Confluence 또는 새로 구매하시는 사용자는 Java 1.8 패치 지원이 중단되면서 구매를 해야하나? 궁금하실 텐데요.

최신 Jira, Confluence Open JDK 를 지원합니다.

물론 Oracle Java도 같이 지원됩니다.

https://confluence.atlassian.com/jirasoftware/jira-software-7-13-x-release-notes-957981568.html#JiraSoftware7.13.xreleasenotes-openjdk

https://confluence.atlassian.com/doc/confluence-6-13-release-notes-959288785.html


현재 1.8.181로 사용하고 계시다면 Jira, Confluence 최신 버전이 안정화 될 때 까지 지켜보다가 OpenJDK로 운영하시는게 좋을 것 같습니다.

오픈소스컨설팅과 함께면 더욱 좋겠죠?

Bundle로 제공되는 JRE는?

Jira installer로 설치하면 JRE 1.8.0_181이 기본으로 제공됩니다.

하지만 보안, 성능 다양한 이슈로 1.8.0_181 을 계속 사용할 수 없겠죠.

이에 관련해서는 아래 링크에서 Road map 지속 적인 확인이 필요할 것 같습니다.

https://community.atlassian.com/t5/Jira-discussions/Java-11-and-OpenJDK-support-for-Atlassian-Server-amp-Data-Center/td-p/872998

Jira는 adoptOpenJDK를 지원 한다고 하네요. https://adoptopenjdk.net/

Atlassian은 빠르고 유연하게 Java 라이선스 문제를 해결하고 있습니다.

글로벌 공룡 기업의 정책에 따라서 개발 문화, 기업 문화가 바뀌는 것 같습니다.

우리가 도구, 개발언어, 기술 스택을 선택할 때 신중해야 하는 이유입니다.

감사합니다.
