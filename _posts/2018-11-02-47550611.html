안녕하세요? #오픈소스컨설팅 #아틀라시안 흥부자 김세연입니다.

#오픈소스컨설팅 은 #Atlassian User Group #AUG 공식 서포터로 활동하고 있습니다.

(많이들 오세요!! 몸만 오시면 됩니다!! )

오늘은 강승현 님의 50명 이하 조직의 Jira Cloud 도입과정에 대해서 발표를 하셨습니다.

승현 님이 다니는 Ecube Labs 회사는 스마트 쓰래기통을 만드는 일을 하고 계신데요

하드웨어, 소프트웨어, 영업 모두 하는데 충돌이 많았데요. 그걸 정리하기 위해서 Jira를 도입 하셨다고 합니다.

처음에는 동아리로 시작하셨는데 우연히 투자를 받았습니다. 초기 프로세스가 없었고, 자율성은 높으나 공유, 소통이

어려웠다고 하네요...

그리고 자료는 개인 컴퓨터에 보관이 되고 인수인계가 매우 어려웠다고 합니다.

구성원의 생각을 공유하고 문제점을 공유하기 위해서 Jira를 도입했다고 합니다.

Jira 도입 과정

  • 유명한 Issue Tracker
  • Jira Cloud를 선택한 이유
    • Cloud에 대한 인식이 좋았음
    • 설치와 업데이트를 신경 쓸 필요가 없다.
    • 혼자 관리중인 솔루션이 10개가 넘어서 관리 포인트 줄이고 싶었다.
  • 전사에 도입하기 전에 우리 팀만 써보자!
    • 팀장님을 설득해서 결재하고!
    • SW 엔지니어들이라 적응도 빠르고 필요성을 느끼고 있었기 때문에 일단 만족도 짱
    • 모든 업무 하나하나를 이슈로 만들어서 사용
    • 한 달 동안 SW팀은 정말 잘 사용했고 만족도가 높았음
    • 한 달 후 워크숖에서 승현 님께서 Jira 전사 도입을 추진해서 됐음!!
  • 전사 도입 후 어떻게 사용했을까!
    • 회사에 존재하는 역할에 따라 프로젝트를 구성 (수 많은 프로젝트가 생김...)
    • 프로젝트가 엄청나게 생겨나기 시작했음!! 관리가 안되기 시작함
    • 그래서 프로젝트를 성격에 따라 구성하고 나누기 시작했음
    • 영업같은 경우는 미국, 중국, 유럽 등 나뉘어서 만들어 달라는 요청이 있어 만들어 줬음 (앞으로 하나의 프로젝트로 관리 예정)

도입 하면서 느낀 것...

  • Jira를 도입하면서 어려운 업무도 있다..
    • 스마트 쓰레기통을 수리하는 업무가 있는데 대부분 밖에서 일하는 사용자는 Jira를 사용하기 어려움
      • 코치도 어렵고, Jira 접근성도 떨어짐
    • Jira를 사용하기 힘든 부서가 있다. 모든 업무가 Jira에 어울리는 것은 아니다.
  • 프로젝트는 무조건 나눈다고 좋은게 아니었다..
    • 하나의 프로젝트에 업무 별로 나눈면 좋지만.. JQL은 사용자 친화적이지 않다고 느낌.
    • 사용자들은 필터를 만들기 힘들어하고 UI가 친숙하지 않다.

Cloud 서비스 장/단점

  • 장점: 자동 업데이트
  • 단점: 자동 업데이트
  • 무엇!? 업데이트가 너무 빠르다보니 사용자들이 적용하기 힘들어하고 버그가 있음
    • "갑자기 이 화면에 있었는데 어디갔어? 새로운 기능은 이거 어떻게 해?" 이런 것들이 있음
  • 자동 업데이트 이후 일부 플러그인 사용에 문제가 발생
    • 플러그인 개발사 OR Atlassian이 지원해줄 때 까지 "이거 왜 갑자기 안되요?"에 시달려야 했다..
  • 비교적 비싸다!
  • 느리다..
  • 아직 많이 불안정하다
    • 파일 업로드가 자주 실패하는 것은 물론, 안되는 경우도 있다.
    • 이미 업로드 된 파일 접근에 실패하여 몇번이나 재시도를 해야하는 경우가 비일비재
  • 도메인 변경이 안된다.
  • Storage 가 너무 적다. 25GB
  • Cloud에서 Server로 마이그레이션이 불안정하다고 함!

결론

  • Jira 보다는 Confluence 더 예쁘고 문제 없고 깔끔하다.
  • Jira Cloud도 좋아지고 있다.
  • Cloud 버전을 회사에 도입 하려면 조금 더 기다려보자.

이후에 질문과 답변 시간을 가졌습니다.

  • Jira Cloud는 대량 변경 기능이 없다!?  없다네요..
    • 프로젝트를 정리하는데 어려움이 있다고 합니다. ㅠ
    • Github은 Create Branch 기능이 없다. Bitbcuket만 있음
  • Jira를 사용하면서 개발 문화가 어떻게 변경 됐는지?
    • PR 문화가 정착 되어 코드 리뷰를 잘 하고 있고 개발자들 만족도가 높음.
  • Google Drive 랑 Confluence같이 사용하고 있는데 교통 정리는 어떻게 하고 있는지?
    • 최근 문서는 Confluence에 작성하고 있고 영업팀은 Export문제로 Google Drive 를 사용하고 있음
  • 계정은 G Suite를 사용하고 있음
  • Jira 사용을 이끌면서 싫어하는 팀도 있었을 텐데 시작하게 된 이유?
    • 일단 일을 똑똑하게 편하게 하기 위해서 시작했고, 워크샾 때 희망하는 팀은 팀장을 통해서 접수를 받았고 꾀 많이 접수를 해주셨음
  • 메신저는 S/W팀은 Slack을 사용하고 업무팀은 라인을 사용하고 있음
